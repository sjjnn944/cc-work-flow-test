# DLP (Data Loss Prevention) 보안 솔루션 요구사항

DLP 솔루션의 일반적인 기능을 항목별로 정리한 문서.

---

## 1. 데이터 식별 및 분류 (Data Discovery & Classification)

### 1.1 콘텐츠 검사 (Content Inspection)
- **키워드 매칭**: 사전 정의된 키워드/패턴 기반 민감 데이터 탐지
- **정규표현식 매칭**: 주민등록번호, 카드번호, 계좌번호 등 정형화된 패턴 탐지
- **데이터 핑거프린팅**: 원본 문서의 지문(fingerprint)을 생성하여 파생 문서 탐지
- **머신러닝 기반 분류**: 자연어 처리(NLP)를 통한 비정형 데이터 자동 분류
- **OCR (광학 문자 인식)**: 이미지/스캔 문서 내 텍스트 추출 및 검사

### 1.2 데이터 분류 체계
- **보안 등급 분류**: 극비, 비밀, 대외비, 일반 등 등급별 분류
- **자동 분류**: 콘텐츠 분석을 통한 자동 보안 등급 부여
- **수동 분류**: 사용자가 직접 문서 등급을 지정하는 기능
- **분류 라벨링**: 문서 헤더/푸터/워터마크에 보안 등급 자동 삽입
- **분류 정책 관리**: 조직별/부서별 분류 기준 정의 및 관리

### 1.3 개인정보 탐지
- **주민등록번호** (13자리)
- **여권번호**, **운전면허번호**
- **신용카드번호** (PCI-DSS 대응)
- **계좌번호**, **전화번호**, **이메일 주소**
- **건강정보** (HIPAA 대응)
- **고유식별정보** 및 **민감정보** (개인정보보호법 대응)

## 2. 엔드포인트 DLP (Endpoint DLP)

### 2.1 저장 매체 제어 (Device Control)
- **USB 저장장치**: 읽기/쓰기 차단, 허용 목록(화이트리스트) 관리
- **외장 하드디스크**: 연결 탐지 및 차단
- **CD/DVD/Blu-ray**: 쓰기 차단
- **모바일 기기 (MTP/PTP)**: 스마트폰/태블릿 연결 제어
- **블루투스**: 파일 전송 차단
- **적외선 (IrDA)**: 통신 차단
- **Wi-Fi Direct**: P2P 파일 전송 차단
- **장치별 세분화 정책**: 장치 시리얼번호/벤더ID/제품ID 기반 제어

### 2.2 애플리케이션 제어
- **프로세스 기반 제어**: 실행 파일 경로/해시/서명 기반 앱 허용/차단
- **클립보드 제어**: 복사/붙여넣기 시 민감 데이터 차단 (앱 간 전송 포함)
- **화면 캡처 차단**: PrintScreen, 캡처 도구(Snipping Tool 등) 차단
- **원격 데스크톱 제어**: RDP, TeamViewer, AnyDesk 등 원격 접속 시 파일 전송 차단
- **가상머신 제어**: VM 내부-외부 간 데이터 이동 감시

### 2.3 파일 시스템 감시
- **파일 복사/이동/삭제 감시**: 실시간 파일 작업 모니터링
- **파일 암호화 강제**: 지정 경로 내 파일 자동 암호화
- **파일 반출 승인 워크플로우**: 결재 프로세스를 통한 파일 반출 관리
- **파일 원본 보관**: 반출 파일의 사본을 서버에 자동 저장
- **숨김 파일/압축 파일 검사**: 다중 압축, 암호화 압축 파일 내부 검사

### 2.4 출력물 보안
- **프린터 출력 제어**: 로컬/네트워크 프린터 출력 승인/차단
- **출력물 워터마크**: 출력 시 사용자 정보, 일시 등 워터마크 자동 삽입
- **출력 로그 기록**: 출력 문서명, 페이지 수, 출력자, 시간 등 기록
- **출력물 이미지 보관**: 출력된 문서의 이미지를 서버에 저장

## 3. 네트워크 DLP (Network DLP)

### 3.1 이메일 제어
- **첨부파일 검사**: 발신 이메일 첨부파일 내 민감 데이터 탐지
- **본문 검사**: 이메일 본문 내 민감 정보 포함 여부 검사
- **발송 차단/보류**: 정책 위반 시 발송 차단 또는 관리자 승인 대기
- **암호화 강제**: 외부 발송 시 첨부파일 자동 암호화
- **수신자 제한**: 외부/내부 수신자 도메인 기반 제어

### 3.2 웹/클라우드 제어
- **웹메일 차단**: Gmail, Naver Mail 등 웹메일을 통한 파일 업로드 차단
- **클라우드 스토리지 제어**: Google Drive, Dropbox, OneDrive 등 업로드 제어
- **SNS 차단**: 메신저, SNS를 통한 파일 전송 차단
- **웹 업로드 감시**: HTTP/HTTPS POST를 통한 파일 업로드 탐지
- **SSL/TLS 복호화**: HTTPS 트래픽 복호화를 통한 콘텐츠 검사

### 3.3 프로토콜 제어
- **FTP/SFTP**: 파일 전송 프로토콜 감시 및 차단
- **SMB/CIFS**: 네트워크 공유 폴더 파일 전송 제어
- **P2P**: 토렌트 등 P2P 프로토콜 차단
- **메신저 프로토콜**: 카카오톡, Slack, Teams 등 메신저 파일 전송 제어
- **커스텀 포트/프로토콜**: 비표준 포트를 통한 데이터 유출 탐지

### 3.4 네트워크 트래픽 분석
- **DPI (Deep Packet Inspection)**: 패킷 레벨 콘텐츠 분석
- **프로토콜 이상 탐지**: 비정상 프로토콜 사용 탐지
- **데이터 유출 패턴 분석**: 비정상적 대량 전송, 시간대 이상 등 분석
- **DNS 터널링 탐지**: DNS를 통한 데이터 유출 시도 탐지

## 4. 스토리지 DLP (Storage DLP / Data at Rest)

### 4.1 데이터 검색 및 스캔
- **파일 서버 스캔**: 네트워크 파일 서버 내 민감 데이터 탐지
- **데이터베이스 스캔**: DB 내 개인정보/민감정보 탐지
- **엔드포인트 로컬 스캔**: 사용자 PC 내 민감 파일 검색
- **클라우드 스토리지 스캔**: 클라우드에 저장된 민감 데이터 탐지
- **정기 스캔 스케줄링**: 자동화된 주기적 스캔 수행

### 4.2 데이터 보호 조치
- **자동 암호화**: 탐지된 민감 파일 자동 암호화
- **격리/이동**: 민감 파일을 보안 저장소로 자동 이동
- **접근 권한 변경**: 파일 권한을 자동으로 제한 설정
- **삭제**: 정책에 따라 비인가 민감 데이터 자동 삭제
- **알림**: 데이터 소유자/관리자에게 민감 데이터 발견 알림

## 5. 정책 관리 (Policy Management)

### 5.1 정책 구성
- **규칙 기반 정책**: 조건(who, what, where, when, how) 기반 세밀한 정책 설정
- **사용자/그룹 기반 정책**: AD/LDAP 연동, 조직도 기반 정책 적용
- **예외 정책**: 특정 사용자/부서/상황에 대한 예외 규칙 설정
- **시간 기반 정책**: 근무시간/비근무시간 차등 정책
- **위치 기반 정책**: 사내/사외(VPN) 환경별 차등 정책

### 5.2 정책 템플릿
- **규제 준수 템플릿**: 개인정보보호법, GDPR, HIPAA, PCI-DSS 등
- **산업별 템플릿**: 금융, 의료, 제조, 공공 등 산업별 사전 정의 정책
- **커스텀 템플릿**: 조직 고유 정책 템플릿 생성 및 관리

### 5.3 정책 배포
- **중앙 집중 배포**: 관리 서버에서 전체 에이전트에 일괄 배포
- **점진적 배포**: 부서/그룹별 단계적 배포
- **정책 버전 관리**: 정책 변경 이력 관리 및 롤백
- **오프라인 정책 캐싱**: 네트워크 단절 시에도 마지막 정책 유지

## 6. 모니터링 및 로깅 (Monitoring & Logging)

### 6.1 실시간 모니터링
- **대시보드**: 전체 보안 현황을 실시간으로 표시하는 통합 대시보드
- **실시간 알림**: 정책 위반 즉시 관리자 알림 (이메일, SMS, 팝업)
- **사용자 활동 모니터링 (UAM)**: 사용자별 파일 작업/웹 활동/앱 사용 추적
- **위험 사용자 추적**: 위반 빈도가 높은 사용자 별도 모니터링

### 6.2 로그 관리
- **이벤트 로그**: 모든 DLP 이벤트(차단, 허용, 경고) 기록
- **감사 로그 (Audit Log)**: 정책 변경, 관리자 활동 등 감사 추적
- **파일 작업 로그**: 파일 생성/복사/이동/삭제/출력 이력
- **네트워크 전송 로그**: 파일 전송 이력 (발신자, 수신자, 파일 정보)
- **로그 보존 정책**: 법적 요구사항에 따른 로그 보존 기간 설정
- **로그 무결성 보장**: 로그 위변조 방지 (해시, 타임스탬프)

### 6.3 보고서
- **정기 보고서**: 일간/주간/월간 보안 현황 보고서 자동 생성
- **규제 준수 보고서**: 법적 감사 대응용 보고서
- **사고 보고서**: 보안 사고 발생 시 상세 보고서
- **통계 보고서**: 위반 유형별, 부서별, 기간별 통계 분석
- **커스텀 보고서**: 사용자 정의 조건에 따른 보고서 생성

## 7. 사고 대응 (Incident Response)

### 7.1 사고 탐지 및 분류
- **자동 사고 생성**: 정책 위반 시 자동으로 보안 사고 티켓 생성
- **심각도 분류**: 위반 유형/데이터 등급에 따른 자동 심각도 분류 (Critical/High/Medium/Low)
- **오탐 관리**: 오탐(False Positive) 마킹 및 정책 자동 튜닝
- **사고 연관 분석**: 동일 사용자/동일 패턴의 반복 사고 연관 분석

### 7.2 대응 워크플로우
- **자동 대응**: 차단, 격리, 암호화 등 자동 대응 조치
- **에스컬레이션**: 심각도에 따른 자동 상위 보고 체계
- **승인 프로세스**: 파일 반출/예외 처리 시 다단계 결재
- **사고 조사 도구**: 타임라인 분석, 관련 이벤트 추적, 포렌식 데이터 수집

### 7.3 사용자 알림
- **실시간 팝업 알림**: 정책 위반 시 사용자에게 즉시 경고
- **교육 연계**: 위반 시 관련 보안 교육 콘텐츠 안내
- **사유 입력 요청**: 반출 시도 시 사유 입력 강제

## 8. 관리 및 운영 (Administration)

### 8.1 관리 콘솔
- **웹 기반 관리 콘솔**: 브라우저를 통한 중앙 관리
- **역할 기반 접근 제어 (RBAC)**: 관리자 권한 세분화 (슈퍼관리자, 부서관리자, 모니터링 전용 등)
- **멀티 테넌트**: 다중 조직/부서별 독립 관리 지원
- **관리자 활동 감사**: 관리자의 모든 작업에 대한 감사 로그

### 8.2 에이전트 관리
- **원격 설치/제거**: 중앙에서 에이전트 원격 배포 및 제거
- **자동 업데이트**: 에이전트 버전 자동 업데이트
- **에이전트 상태 모니터링**: 설치 상태, 동작 상태, 버전 정보 모니터링
- **에이전트 자기 보호**: 에이전트 프로세스 강제 종료/삭제 방지
- **에이전트 무결성 검증**: 에이전트 파일 위변조 탐지

### 8.3 시스템 연동
- **Active Directory / LDAP**: 사용자/그룹/조직도 동기화
- **SIEM 연동**: Splunk, QRadar, ArcSight 등 SIEM 시스템에 로그 전송
- **메일 서버 연동**: Exchange, Gmail 등 메일 서버와 연동
- **프록시/방화벽 연동**: 네트워크 장비와의 연동
- **SOAR 연동**: 자동화된 사고 대응 플랫폼과의 연동
- **API 제공**: RESTful API를 통한 외부 시스템 연동

## 9. 성능 및 안정성 (Performance & Reliability)

### 9.1 성능 요구사항
- **에이전트 리소스 사용 최소화**: CPU 5% 이하, 메모리 100MB 이하 (유휴 시)
- **네트워크 부하 최소화**: 정책/로그 전송 시 대역폭 최적화
- **검사 속도**: 파일 검사 시 사용자 작업 지연 최소화 (체감 지연 1초 미만)
- **대규모 환경 지원**: 10,000대 이상 엔드포인트 동시 관리

### 9.2 안정성 요구사항
- **고가용성 (HA)**: 관리 서버 이중화, 장애 시 자동 전환
- **장애 복구**: 서버 장애 시 에이전트 독립 동작 (오프라인 모드)
- **데이터 백업**: 정책, 로그, 설정 자동 백업 및 복원
- **에이전트 안정성**: BSOD 방지, 시스템 충돌 시 자동 복구

---

## 10. 보안 및 규제 준수 (Security & Compliance)

### 10.1 솔루션 자체 보안
- **관리자 인증**: 다중 인증(MFA) 지원
- **통신 암호화**: 에이전트-서버 간 TLS 1.2+ 암호화 통신
- **데이터 암호화**: 저장 데이터(로그, 정책) AES-256 암호화
- **접근 제어**: 관리 콘솔 IP 제한, 세션 타임아웃

### 10.2 규제 준수 지원
- **개인정보보호법**: 개인정보 처리 단계별 보호 조치
- **정보통신망법**: 정보통신서비스 제공자 보안 요구사항
- **GDPR**: EU 개인정보 보호 규정 대응
- **PCI-DSS**: 카드 결제 데이터 보안 표준
- **HIPAA**: 의료정보 보안 규정
- **ISO 27001**: 정보보안 관리체계 인증 대응

## 11. 배포 및 호환성 (Deployment & Compatibility)

### 11.1 지원 환경
- **Windows**: Windows 10, 11, Server 2016/2019/2022
- **macOS**: 최신 3개 버전 지원
- **Linux**: Ubuntu, CentOS, RHEL (서버 환경)
- **가상 환경**: VDI (Citrix, VMware Horizon), 가상 서버

### 11.2 배포 방식
- **On-Premise**: 사내 서버 설치형
- **Cloud (SaaS)**: 클라우드 기반 서비스형
- **Hybrid**: On-Premise + Cloud 혼합 구성
- **MSP 지원**: 관리형 보안 서비스 제공자를 통한 운영

### 11.3 확장성
- **수평 확장**: 서버 추가를 통한 처리 용량 확장
- **모듈형 구조**: 필요한 기능만 선택적 도입
- **라이선스 모델**: 사용자 수/디바이스 수 기반 유연한 라이선스
